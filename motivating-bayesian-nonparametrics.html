Motivating Bayesian Nonparametrics

Date Sun 18 January 2015 Tags ml

Remark: From the first part of a talk by Dr. Zoubin Ghahramani at NIPS 2011. Also, from UAI 2005 tutorial from the same author.

Why

  • Bayesian? Because it is a simple framework. Model based methods are relatively more intuitive.
  • Nonparametrics? Because the real word is complex. Bayesian models are powerful only when prior captures our beliefs well. Inflexible models yield unreasonable inferences.

Bayesian machine learning

  • Almost everything in this approach follows from two rules: the sum rule (

    $$P(x) = \sum_y P(x,y)$$
    ) and the product rule (
    $$P(x,y) = P(x\|y)P(y)$$
    ).

  • Let

    $$P(\theta)$$
    be the prior and
    $$P(D\|\theta)$$
    be the likelihood of
    $$\theta$$
    and let
    $$P(\theta\|D)$$
    be the posterior. Here
    $$\theta$$
    is the model parameter random variable and
    $$D$$
    is the data. We are interested in the posterior distribution given data.

  • We can do prediction as

    $$P(x\|D) = \int P(x\|\theta,D)P(\theta\|D) d\theta$$
    using the posterior we compute. Our prediction is a conditional distribution that averages over all
    $$\theta$$
    values.

Parametric (P) vs Nonparametric (NP)

  • P

  • Assumes a finite set for

    $$\theta$$
    .

  • Also,

    $$P(x\|\theta,D) = P(x\|\theta)$$
    . That is,
    $$\theta$$
    captures everything about the data. Hence, the complexity of modeling is bounded even if data becomes unbounded. Ultimately, P is not flexible.

  • NP

  • Assumes that data distribution cannot be defined using a finite set of parameters. Need infinite dimensional

    $$\theta$$
    . Think of
    $$\theta$$
    as a function.

  • Increasing amount of info can be captured by
    $$\theta$$
    as data grows.

Two building blocks for Bayesian NP

  • Gaussian Process (GP)

  • A GP defines a distribution

    $$P$$
    over real-valued functions (
    $$f \sim GP(\cdot\|\mu,c)$$
    ). We can think of GP as an infinite dimensional Gaussian.

  • A measure on functions is a GP if for any finite subset of the domain, the marginal distribution over that finite subset

    $$P([f(x_1),...,f(x_n)]$$
    has a multivariate Gaussian distribution.

  • It is parameterized by the mean and covariance functions.

  • Once we use this as a prior in say some regression problem, then the posterior measure on the set of functions can be derived from the usual sum and product rules.

  • Dirichlet Process (DP)

  • A DP defines a distribution on distributions (

    $$G \sim DP(\cdot\|G_0,\alpha)$$
    ) with
    $$G_0$$
    as the base measure (over a measurable space
    $$\Theta$$
    ) and
    $$\alpha > 0$$
    as the concentration parameter. Think of it as an infinite dimensional Dirichlet distribution or as a non-negative function that integrates to one.

  • A Dirichlet distribution is a measure over the

    $$K$$
    dimensional simplex. It is conjugate to the multinomial distribution. So if the likelihood of data given a probability vector (belonging to the
    $$K$$
    -simplex), then the posterior measure on the simplex is again Dirichlet.

  • Back to DP. For every finite partition

    $$(A_1,...,A_K)$$
    of a measurable space
    $$\Theta$$
    , if
    $$G$$
    is a DP, then
    $$(G(A_1),...,G(A_K)) \sim\textrm{Dir}(\alpha G_0(A_1),...,\alpha G_0(A_K))$$
    .

  • The process has some interesting properties. One of them is that a realization or sample from DP is a discrete measure 'with probability one'. It also has an interesting conjugacy property. If the prior on

    $$G$$
    is DP and we observe n examples from the space
    $$\Theta$$
    sampled according to
    $$G$$
    , then the posterior is also DP (this is proved using Dirichlet-Multinomial conjugacy).

  • Both these have some interesting 'coverage' properties in the sense that, for any function in a large set of functions, there is a probability mass on a suitably nearly function due to these processes.

Other items on the menu to be aware of

  • Dirichlet process mixture, Pitman-Yor diffusion tree, Indian buffet process, Mondrian process, exchangeability, Gibbs sampling, Expectation Propagation (EP).

Closing thoughts

  • Sporadic success stories in motion capture segmentation, word segmentation, language modeling.

  • Bayesian machine learning treats learning as an inference problem (contrast this with parameter estimation view). It seems to work well when proposed models are flexible and capture data properties.

  • Have to justify approximate inference else we cannot defend 'Bayesian coherence'.

  • Model based machine learning is not always better. See LLE, spectral clustering etc.

  • Structure discovery (as in number of clusters in clustering) is fragile.

  • NP methods are computationally expensive (for many inference problems).

  • Large P modeling techniques (notably Deep neural nets) perform well in practice. Is NP just a question of elegance?
