Deep Learning 101

Date Thu 20 March 2014 Tags math

Linear Chain CRFs and Inference:

  • Note: I'll use the notation from the previous post on feed-forward neural net training.
  • Conditional Random Fields (CRFs) are undirected graphical models. Different from HMM and Bayes Nets.
  • Think of Conditional Random Fields in the context of sequence classification. Here, examples are organized in a sequence.
  • There is information in the sequence (unlike the permutation invariance of ERM where we can assume i.i.d examples)

  • Look at the following feed-forward NN operations which happen in a sequence:

    \begin{eqnarray*} x^{t-1} &\rightarrow& a^{L+1}(x^{t-1}) \rightarrow p(y^{t-1}\| x^{t-1})\\ x^{t} &\rightarrow& a^{L+1}(x^{t}) \rightarrow p(y^{t}\| x^{t})\\ x^{t+1} &\rightarrow& a^{L+1}(x^{t+1}) \rightarrow p(y^{t+1}\| x^{t+1}) \end{eqnarray*}

  • Exploit dependence of labels. Model

    $$p(y^1,...,y^T \| x^1,...,x^T)$$

  • New notation: (Uppercase)
    $$(X^t,Y^t)$$
    are sequence pairs.
    $$X^t = [x_1^t,...,x_{K_t}^t]$$
    and
    $$Y^t = [y_1^t,...,y_{K_t}^t]$$
    .
  • Linear chain CRF:
  • Regular classification:
    $$p(Y\|X) = \Pi_{k}^{K} p(y_k\|x_k) = \Pi_{k}^{K} \exp(a^{L+1}(x_k)_{y_k} / Z(x_k))$$
  • Linear chain sequence classification:
    $$p(Y\|X) = \frac{1}{Z(X)}\exp(\sum_{k}^{K} a^{L+1}(x_k)_{y_k} + \sum_{k}^{K} V_{y_k,y_{k+1}}) $$
  • Notice the term
    $$V_{y_k,y_{k+1}}$$
  • $$Z(X)$$
    is called the partition function
  • Can already see intractability
  • Same weights W and V matrix
  • Context window (say of size 1):
    - separated feed: $$p(Y\ X) = \frac{1}{Z(X)}\exp(\sum_{k=1}^{K} a^{L+1,0}(x_k){y_k} + \sum^{K} V_{y_k,y_{k+1}} + \sum_{k=2}^{K}a^{L+1,-1}(x_{k-1}){y_k} + \sum^{K-1}a^{L+1,+1}(x_{k+1})_{y_k} ) $$
  • unary log factor:
    $$a_{u}(y_k)$$
  • pairwise log factors:
    $$a_p(y_k,y_{k+1})$$
  • Then,
    $$p(Y\|X) = \exp(\sum_{k=1}^{K} a_u(y_k) + \sum_{k=1}^{K-1}a_p(y_k,y_{k+1}) )/Z(X)$$
  • Computing
    $$Z(X) = \sum_{y_1}\sum_{y_2}\cdots\sum_{y_K} \exp(\sum_{k=1}^{K} a_u(y_k) + \sum_{k=1}^{K-1}a_p(y_k,y_{k+1}) ) $$
    requires a sum of exponential terms
  • We use linear structure to come up with a dynamic program which can do this polynomially.
    • Reorganize terms.
    • $$Z(X) = \sum_{y_{K}} \exp(a_u(y_K))\left( \sum_{y_{k-1}}\exp(a_u(y_{K-1}) + a_p(y_{K-1},y_K)) \cdots \left(\sum_{y_1}\exp(a_u(y_1) + a_p(y_1,y_2)) \right)\cdots \right) $$
    • Let
      $$\alpha(y_2) = \sum_{y_1}\exp(a_u(y_1) + a_p(y_1,y_2))$$
    • Get nested sums like above.
    • Total calculation takes
      $$K*\textrm{support}^2(y_k)$$
      .
    • Could have done reverse nesting (sum over the last label, then proceed to the next to last label, etc...).
    • Call these
      $$\beta_{K}(y_{K-1}), \beta_k(y_{k-1})$$
      ...
    • Forward Backward algorithm for CRFs === sum-product === belief-propagation for linear chains.
    • $$\{\alpha,\beta\}$$
      can be used for other interesting outputs as well.
    • Better to work in the log-space for numerical stability
    • log-sum-exp may still be numerically unstable. Use trick
      $$log(\sum \exp(z_i)) = \max_i(z_i) + \log\sum\exp(z_i - \max_i(z_i))$$
  • Computing marginals in linear chain CRFs (a.k.a. inference):
    • Compute
      $$\alpha, \beta$$
      tables.
    • Say we want to compute
      $$p(y_k\|X), p(y_k,y_{k+1}\|X)$$
    • We can derive the following result:
      $$p(y_k\|X) \propto \exp(a_u(y_k) + \log\alpha_{k-1}(y_k) + \log\beta_{k+1}(y_k))$$
      . The pairwise marginal is derived in a similar manner.
  • Classification using linear chain CRFs:
    - Option 1: at each position
    $$k$$
    , pick the label
    $$y_k$$
    with highest marginal probability $$p(y_k\    		 X)$$ (this strategy corresponds to minimizing the expected sum of errors at each position in the sequence)
  • General CRF Terminology: factors, sufficient statistics, ...
    • CRF standard form:
      $$p(Y\|X) = \frac{1}{Z(X)} \Pi_{f}\Psi_{f}(Y,X)$$
      where
      $$f$$
      indexes factors.
    • The factor can be parametrized using sufficient stats
      $$\{t_{f,s}(Y,X)\}$$
      :
      $$\Psi_{f}(Y,X) = \exp(\sum_{s}\theta_{f,s}t_{f,s}(Y,X))$$
    • linear CRF: where there is no hidden layer, the input is mapped to output in a linear fashion
  • Markov Network:
    • Network (undirected!) with an edge between each node (random variable) that shares a factor in the factor representation of CRF
    • Easily identify conditional : Two nodes are conditionally independent if all paths between them contain at least one of the conditioning nodes
  • Undirected vs directed:
    • Only need factors
      $$\geq 0$$
      for undirected graphical models like CRFs.
    • Factors also need to sum to 1 with respect to the appropriate argument for directed graphical models like HMM. Need D-separation rule to identify conditional independence.
  • Factor graphs:
    • Graph can have more than one factor decomposition
    • Factor graph: add nodes in the graph corresponding to factors. Provides more information. Use the original Markov net if only care about conditional independence. Else, stick to factor graphs.
